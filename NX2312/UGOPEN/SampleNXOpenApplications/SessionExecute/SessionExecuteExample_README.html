Session.Execute

Description: This example demonstrates how to use the Session.Execute API.

This API allows users to call from one NXOpen application into another NXOpen application (regardless of languange). This can be from the same language into the another binary in the same langauge. Or this can be from one language to another language. For example, one could call from a C++ NXOpen application into another C++ NXOpen application. Or it can be calling from C++ NXOpen into another supported NXOpen Binary such as Java.

This topic is not not considered a beginnering topic to NXOpen development. So it is assumed the reader has some experience with the NXOpen API as well as has knowledge of how to develop in the relevant languages. And the instructions for this example assume the reader is familiar with how to build and run with the relevant lanaguages.

Another assumption is this example will only be run on windows. As we will be using the File Execute dialog to pass in paramters and run the Executor program. This example can be modified to run on Linux, but this excercise is left to the reader.

Layout of Example

There are two seperate categories of automation for this example. The first is "Executor" programs will call into a NXOpen application in each of the languages. The second category is the "To Be Executed" program which will be called by the "caller" programs.

Building the C++, C#, and Java programs.

This example will assume all the programs will be built. It is highly recommended to use the NXOpen Templates to set up the projects for C++ and C#. Just be aware that the executor examples expect the binaries to be built with specified names. If you use a different name, you will need to alter the name of the binary generated in the Excutor programs. Additionally, the C++ code needs to be compiled with C++ 17 standard. As of 2306 this is not what is set by default in the NXOpen Template. In the future NXOpen may increase the the floor C++ version and that should be used. This comment is only to call out until the C++ template is C++ 17 or higher, the user will need to set this manually.

Lastly all the To Be Executed Programs are expected to be collocated, so the Executor program can execute it.

What the Example does?

Each Executor program will take in a directory as a paramter which will contain the To Be Executed programs. And then for each program there it will execute each To Be Executed program, and pass it a list of parameters that consist of a integer, a double, a boolean, a string, and TaggedObject (which will be the work part). Each To Be Execute program will write the listing window the parameters passed in.

How to Run the Example

  1. Start NX.
  2. Create a part. As the work part will be passed to each To Be Executed Program.
  3. Go to File -> Execute ->NX Open.
  4. Navigate to the location of the Executor to be executed and select it
  5. Enter User Function Arguments
  6. In the dialog box that comes up, put in the directory where all the To Be Executed Programs reside. Hit Ok.
  7. Hit ok.
